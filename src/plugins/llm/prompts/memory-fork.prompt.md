# 长期记忆维护任务

下面这段是离线反思指令，**不要回复对话内容、不要扮演角色**。基于以上对话记录，决定是否更新这位用户的长期记忆档案。

**这是一次离线反思**——不是和用户对话，没有人会看到你的输出。请先在内心独白中权衡每条候选信息再决定写什么。

## 输出格式

按以下决策选择**唯一**一种产出：

1. **本次对话没有值得新增的内容，且现有档案符合下文「不该记」规则** → 输出 `{{NO_UPDATE_MAGIC}}`，不要输出其他任何字符
2. **本次对话没有值得新增的内容，但现有档案里有按规则**不该被记忆**的旧条目** → 输出**精简后**的完整档案（仅删除违规旧条目，不新增内容）
3. **有值得新增的内容**（无论旧档案是否需要清理） → 输出**更新后**的完整档案

输出档案时使用 markdown 格式，**直接覆盖**现有档案，不要包裹代码块、不要前后缀、不要解释。

## 容量规则

- 目标：≤ {{SOFT_LIMIT}} 字节（UTF-8）
- 硬上限：{{HARD_LIMIT}} 字节，超出会被拒绝并要求重写
- 容量是硬约束。空间不够就**取舍**——遗忘不重要的旧信息为新信息让路

## 该记什么（按优先级，从高到低）

1. **用户身份与长期偏好**：昵称、自我认同、爱好、口味、价值观、回复风格倾向
   - 例：`喜欢吃热干面`、`偏好简洁回复`、`是前端工程师`
2. **跨会话重要事项**：未解决的问题、正在追踪的事、明确表达过的请求
   - 例：`下周要参加 PyConf`、`让我把所有插画图保存到 D 盘`
3. **互动模式与雷区**：用户在乎什么、讨厌什么、什么话题敏感
   - 例：`不喜欢被纠正语法`、`忌讳被叫"老板"`

## 绝对不要记的（即使对话里出现了）

- ❌ **本次会话的瞬时事件**：测试过什么指令、调用工具的结果、查到的某个具体值
  - 例（错误）：`掷骰子结果 14`、`询问过支持的工具`、`测试了错误处理`
- ❌ **任务进度日志**：完成了什么、解决了什么 bug
- ❌ **你能立刻重新发现的事**：通用知识、可以再问一次的信息
- ❌ **基于一两次对话就能推断的"性格判断"**：避免主观臆测
- ❌ **不影响未来对话的细节**：聊天本身的痕迹不该被记忆

如果一条信息只能在"它**今后几次**还会被用到"才有价值，而你不确定，就**不要写进去**。

## 取舍准则

写之前问自己三个问题：

1. **这条信息一周后还有意义吗？** 若没有，跳过。
2. **下次对话遇到同样情形，知道它能让我回复得更好吗？** 若不能，跳过。
3. **如果空间紧张，我会选择保留这条还是别的？** 排序后只留高优先级。

如果三问之后都没东西好留，**直接输出 `{{NO_UPDATE_MAGIC}}`** 是最优解。**不要为了"必须有产出"而硬塞条目**——空记忆比烂记忆好。

## 顺手清理旧记忆

每次反思时，**顺便审视**当前档案里的每一条：

- 这条是不是上面「绝对不要记」清单里的某种？（例：单次工具调用结果、已结束的测试性对话）
- 这条信息现在还有意义吗？或者已经过时？

如果发现违规或失效的旧条目，即便本次对话没有新内容，也应该输出**精简后的完整档案**（输出格式的第 2 种情形），把违规条目去掉。**不要保留明显不该记的旧条目**——别人不会替你修，只能由你下次反思时清理。

但也**不要无端重写**：如果旧档案的每条都符合规则、只是表述不够漂亮，**保持原样**，输出 `{{NO_UPDATE_MAGIC}}`。语言风格的微调不值得占用更新预算。
